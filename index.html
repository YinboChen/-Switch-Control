About this app

  1. Instructions for using the app

    This app provides four versions which can be controlled with one key, two keys, three keys and five keys.

    In the one key version

    • Each button on calculator will be selected automatically.
    • You can using "q" key to input the selected item for calculation.
    • You can input time as a parameter to change cycle rate.

    In the two keys version

    • You can using "q" key to input the selected item for calculation.
    • Using "d" key to select the next item.

    In the three keys version

    • Using "q" key to input the selected item for calculation.
    • Using "d" key to select the next item.
    • Using "a" key to select the previous item.

    In the five keys version

    • Using "q" key to input the selected item for calculation.
    • Using "d" key to select the next item.
    • Using "a" key to select the previous item.
    • Using "w" key to select the above item.
    • Using "s" key to select the below item.

  3. Data analysising

    Comparing the one key, two keys, three keys and five keys version, it shows that operation by hand spent the less time

    than operation by others such as using mouth pen in these experiments. It took the least amount of time only 23.16 s to

    completed the calculation when using five keys version by hand. When using the two keys version and three keys versions

    by hand, the operating time is similar, but comparing the data with mouth pen, it shows that the two keys version took the

    longest time, significantly longer than the three keys version which using the mouth pen. So according to the data,

    it should avoid operating two keys version when using different parts of body as input medium.

    In addition, the operating time of three keys version and five keys version by the mouth pen were similar. It indicted

    that number of keys is not the main factor when using unfamiliar input medium such as the mouth pen in this case.

    Moreover, it’s very interesting to operating the one key version with different speeds. It can’t operate when speed was

    300ms due to excessive error rate, the error rate was almost 100%. When speed was reduced to 500ms, there was still one- fifth

    of the turnover rate. Through the data shows that operating time is proportional to scanning speed. It means that operating

    the one key version needs to trade-off the error rate and scanning speed. Unfortunately, the more accuracy data was not recorded

    in these experiments.
